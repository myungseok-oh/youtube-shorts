# 직장인 생존 백서 - 채널 기획안

## 컨셉
직장인 관심사 연재 채널. 시즌별 주제로 연재감 확보.

## 시즌 1 - 회사 적응 생존편 (입문자 타겟)

| 화 | 주제 | 핵심 내용 |
|----|------|----------|
| 1화 | 첫 출근 생존 가이드 | 첫날 절대 하지 말아야 할 것들 |
| 2화 | 직장 상사 유형 분석 | 꼰대형/방임형/완벽주의형 대처법 |
| 3화 | 점심시간 생존법 | 혼밥 vs 무리 짓기, 눈치 게임 |
| 4화 | 회의실 생존 전략 | 말 잘하는 척 하는 법 |
| 5화 | 칼퇴 vs 야근 눈치 게임 | 칼퇴하고도 욕 안 먹는 법 |
| 6화 | 직장 내 인간관계 지도 | 친해야 할 사람 vs 거리 둘 사람 |

## 연재 포맷

- 제목 공식: "직장인이라면 꼭 알아야 할 [주제] 생존법"
- 영상 길이: 쇼츠 60초 / 본편 3~5분
- 고정 캐릭터: AI 생성 직장인 캐릭터 "김대리" 활용
- 업로드 주기: 주 2~3회 (월/수/금 추천)
- 해시태그: #직장인 #직장생활 #직장인공감 #생존법

## 파이프라인 구현 분석

### 현재 파이프라인과의 차이점

| 항목 | 뉴스 채널 (현재) | 직장인 생존백서 |
|------|-----------------|---------------|
| 대본 생성 | 뉴스 검색 → 팩트 전달 | 시즌/화 번호 기반 → 대본 생성 |
| 주제 선정 | 트렌드/실시간 뉴스 | 미리 정해진 에피소드 테이블 |
| 화자 | 나레이터 1명 | "김대리" 캐릭터 (추후 대화형 확장 가능) |
| 연속성 | 없음 (매회 독립) | 시즌 내 흐름 연결 |
| 톤 | 뉴스 전달 | 공감+유머+실용 |

### 구현 필요 사항

1. **채널 설정 확장** — `content_type: "serial"`, 시즌/에피소드 관리
2. **에피소드 테이블** — 시즌별 주제 목록을 DB 또는 채널 config에 저장
3. **대본 생성 분기** — 뉴스 검색 대신 에피소드 주제+핵심내용을 프롬프트로 전달
4. **에피소드 트래킹** — 다음 화 자동 결정, 이전 화 요약 전달

### 채널 config 예시

```json
{
  "content_type": "serial",
  "speakers": {
    "A": { "name": "김대리", "voice": "ko-KR-Wavenet-C", "personality": "직장 5년차, 현실적 조언" }
  },
  "series": {
    "current_season": 1,
    "current_episode": 1,
    "recap_previous": true
  }
}
```

### 미결정 사항

- 쇼츠 60초만 할지 / 본편 3~5분도 할지
- 나레이터 1명으로 시작할지 / 처음부터 대화형(김대리+후배 등)으로 갈지
- 에피소드 실행 방식: 시즌1 1화부터 순차 자동 vs 매번 에피소드 선택
- 대화형 확장 시 화자별 TTS 음성 매핑 필요 (문장별 speaker 태그 → 음성 분기)
